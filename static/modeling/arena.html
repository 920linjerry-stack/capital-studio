结算榜 · Settlement Board
同一套 deterministic engine，对当前牌堆 — 个 directed pairs 做轻量排序。经济性与现实可行性是两条独立的真相轴，不合并为单一评分。
榜单复用已计算好的结果做本地排序，点选任一行可把它装载回上方牌桌并刷新轻结果卡；需要中密度解释时再打开 Deal Ticket。经济性高不代表可行性稳，反之亦然。
候选交易队列 · Deal Review Queue 0
把感兴趣的并购组合加入本局候选清单：在轻结果卡、Deal Ticket 或结算榜点「加入候选」即可。仅前端会话状态，不写后端/数据库，刷新或换标签页可能丢失。AAPL→MSFT 与 MSFT→AAPL 是两个不同候选，不会混淆方向。
还没有候选交易。在上方轻结果卡、Deal Ticket 或结算榜里点「加入候选」，它们会出现在这里。
点候选行可把该交易重新装载回上方牌桌并刷新轻结果卡；点 ✕ 从候选移除。候选只是本局复盘 shortlist，不进入 EPS / 协同 / 可行性，也不参与双榜排序。
